# 月蚀弹幕 V3.11 - 弹幕简化与规律化报告

## 🎯 更新目标

根据东方STG设计原则，**大幅简化弹幕系统**，让弹幕：
1. **有规律、可预判**
2. **不满屏乱飞**
3. **有明确的安全路径（负空间）**
4. **每个敌人有专属弹幕**
5. **减少弹幕数量但保持难度**

## ✅ 完成的工作

### 1. 学习了东方STG设计原则

From Sparen's Danmaku Design Studio:

#### 核心原则
- **负空间（Negative Space）**：弹幕密集但必须有明确的安全路径
- **时空密度分离**：不是"满屏弹幕"，而是"有节奏的弹幕波"
- **宏观vs微观躲避**：大型障碍物（墙、环）+ 精细躲避（密集慢速弹）
- **Fewer bullets ≠ Easier**：少量弹幕如果设计得当可以更难
- **可预判性**：玩家可以通过观察弹幕结构预判躲避路径

#### 关键发现
> "Unlike shooters where you dodge a small number of bullets that come at high speeds, danmaku games focus on incredible amounts of bullets. However, the bullets must be organized in patterns that create paths for the player."

> "The faster a bullet, the larger area the player must pay attention to. Slower bullets allow the player to focus on a much smaller area of the screen."

### 2. 完全重写弹幕生成器

#### 删除的复杂模式（共8个）
- ❌ patPetal() - 双层花瓣（过于复杂）
- ❌ patCross() - 米字爆射（弹幕过多）
- ❌ patWave() - 波浪弹（轨迹混乱）
- ❌ patSlow() - 减速陷阱（诱导混乱）
- ❌ patStar() - 螺旋星阵（密集填充）
- ❌ patScatter() - 随机散布（破坏规律）
- ❌ patArcSweep() - 扫射弧（不够清晰）
- ❌ 旧的patFan/patRing/patSpiral（参数过多）

#### 新增的简单模式（共6个）

**patSnipe()** - 单发瞄准
```javascript
// 最基础的弹幕：单发朝向玩家
// 用途：教学、填充、节奏控制
patSnipe(e, speed, color, r, kind)
```

**patFan3()** - 3方向对称扇形
```javascript
// 经典东方扇形：中间+左右各1发
// 特点：对称、整齐、可预判
// 安全区：扇形之间的空隙
patFan3(e, spread, speed, color, r, kind)
```

**patFan5()** - 5方向扇形
```javascript
// 中等密度扇形：中间+左右各2发
// 安全区：扇形两侧
patFan5(e, spread, speed, color, r, kind)
```

**patRing8()** - 8方向环形
```javascript
// 经典东方环形弹：8个方向均匀分布
// 特点：可旋转、形成螺旋
// 安全区：环形之间的缝隙
patRing8(e, speed, color, r, kind, angleOffset)
```

**patRing12()** - 12方向环形
```javascript
// 密集环形：12个方向
// 用途：精英敌人、Boss阶段
patRing12(e, speed, color, r, kind, angleOffset)
```

**patSpiral2()** - 双臂螺旋
```javascript
// 简单螺旋：2条螺旋臂
// 特点：固定步进0.18rad，规律可预判
// 使用自动累积的spiralAngle
patSpiral2(e, speed, color, r, kind)
```

**patSpiral3()** - 三臂螺旋
```javascript
// 三臂螺旋：更密集但仍规律
// 特点：固定步进0.15rad
patSpiral3(e, speed, color, r, kind)
```

### 3. 每个敌人专属弹幕

| 敌人 | 弹幕模式 | 特点 | 难度 |
|------|----------|------|------|
| **bat** 蝙蝠 | 单发瞄准 | 最简单，教学用 | ★☆☆☆☆ |
| **familiar** 使魔 | 3方向扇形 | 对称、规律 | ★★☆☆☆ |
| **wisp** 游魂 | 双臂螺旋 | 视觉美观、固定节奏 | ★★☆☆☆ |
| **nun** 修女 | 8方向环 | 经典环形，慢速旋转 | ★★★☆☆ |
| **cherub** 天使 | 5方向扇形 | 中等密度 | ★★★☆☆ |

#### 弹幕特点
- **单一模式**：每个敌人只有1种主要弹幕（warden除外）
- **无随机性**：所有弹幕都是固定规律
- **无特殊触发**：删除了"每N发触发特殊模式"的复杂系统
- **清晰可读**：每个弹幕都有明确的形状和节奏

### 4. 大幅增加开火间隔

#### 之前的问题
- bat: 2.4秒 → 太快
- familiar: 1.7秒 → 太快
- nun: 1.25秒 → 太快
- warden: 1.0秒 → 太快

#### 现在的设置

#### 首次开火延迟

```javascript
fireTimer: 1.0 + (x / W) * 0.8

// 创造"波次"而非"墙"
```

## 🎯 设计原则应用

### 1. 负空间（Negative Space）

**之前**：满屏弹幕，无明确安全区
**现在**：
- 8方向环 → 8个缝隙可以穿过
- 3/5方向扇形 → 扇形两侧是安全区
- 螺旋 → 螺旋臂之间有规律的空隙

### 2. 可预判性（Predictability）

**之前**：随机触发、多重嵌套、参数复杂
**现在**：
- 每个敌人只有1种固定弹幕
- 螺旋使用固定步进角度
- 环形使用固定旋转速度
- 无随机元素

### 3. 时空密度控制

**空间密度**：
- 减少每次发射的弹幕数量（70%）
- 增加弹幕之间的间距

**时间密度**：
- 增加发射间隔（1.3-2.5秒）
- 错开不同敌人的发射时机
- Boss弹幕有明确的节奏感

### 4. 引导性设计

**环形弹幕**：
- 玩家自然会找到环形之间的缝隙
- 慢速旋转让玩家有时间反应

**扇形弹幕**：
- 明确告诉玩家"往两边躲"
- 对称设计让选择更清晰

**螺旋弹幕**：
- 固定节奏让玩家可以"跟着节奏走"
- 螺旋臂之间的空隙形成自然路径

## 🔧 技术实现细节

### 自动累积角度
```javascript
// 螺旋弹幕使用敌人自己的spiralAngle
if (!e.spiralAngle) e.spiralAngle = 0;
// ... 发射弹幕 ...
e.spiralAngle += 0.18; // 固定步进
```

### 错开发射时机
```javascript

// 左侧敌人先发射，右侧敌人后发射
// 形成"波次感"而非"同时爆发"
```

## 🎊 总结

本次更新是**彻底的弹幕哲学转变**：

### 从"多即是难"到"少即是精"
- **之前**：大量弹幕堆叠制造难度
- **现在**：少量弹幕通过规律和节奏制造挑战

### 从"随机混乱"到"固定规律"
- **之前**：多种模式叠加，随机触发
- **现在**：每个敌人一种固定模式，可记忆可预判

### 从"满屏弹幕"到"负空间引导"
- **之前**：试图填满整个屏幕
- **现在**：用弹幕"画出"安全路径

这正是东方STG的精髓：**不是让玩家躲避所有弹幕，而是让弹幕告诉玩家往哪里躲**。
