# 《生命之母》二次落地修改建议与重做指导文档 V2

> 项目：HTML / Canvas / WebGL STG 弹幕射击小游戏  
> 战机：生命之母  
> 文档用途：针对第一版实装效果进行问题诊断、重做要求、代码实现指导、特效分层方案、性能优化和验收标准  
> 适用对象：编程 AI、前端程序员、特效程序、美术整合人员  
> 当前结论：第一版不是“正式游戏特效”，而是“素材叠图预览版”，需要按本文件进行二次落地重做。

---

## 0. 本次重做的核心目标

第一版已经能看到一些方向，但目前存在明显问题：

```text
1. 绿幕素材没有抠干净，能看到方形贴图边界和灰雾覆盖。
2. 大招几乎只是大图叠加，没有代码驱动的动态特效。
3. “生命飘带”被做成粉色细线/激光线，没有飘带质感。
4. 大招连接全屏所有敌人，画面混乱且性能很差。
5. 大招没有明确阶段感，开始、持续、收束、结束爆发混在一起。
6. 贴图层级混乱，大量素材一次性堆叠，导致画面糊、脏、看不清。
7. 战机机制表达不清楚，玩家很难看懂“吸血、回流、回血、爆发”的过程。
```

本次重做不是简单“换几张贴图”，而是要把《生命之母》的核心表现落地为：

```text
一架用生命飘带连接敌人、持续吸血、自损反转回血、最终展开圣域胎宫进行全屏爆发的正式战机。
```

最终视觉目标：

```text
温柔、圣洁、危险、丝带感、生命回流感、晶花绽放感、阶段清晰、不卡顿。
```

# 1. 第一版问题诊断

## 1.1 大招贴图使用错误

当前效果中，大招开启和持续阶段能看到明显问题：

- 大型花朵/领域贴图像整张图直接铺在屏幕上。
- 部分贴图没有正确抠掉绿幕或背景。
- 能看到方形区域、灰蒙蒙的整块贴图、边缘残留。
- 花朵、法阵、晶翼背景与游戏场景没有融合。
- 画面被整张半透明素材盖住，缺少“游戏内特效”的层次。

### 正确要求

所有大招相关贴图必须先预处理成透明 PNG。

```text
禁止：
- 直接把绿幕图整张叠进游戏。
- 直接使用带方形背景的半透明素材。
- 使用一张大图覆盖整个战场来假装大招。
- 让玩家看到贴图原始矩形边界。

必须：
- 抠干净绿幕。
- 删除方形背景。
- 去除绿色/灰色脏边。
- 将花、法阵、光环、粒子拆成独立透明元素。
- 用代码控制缩放、旋转、透明度、闪烁、粒子和冲击波。
```

---

## 1.2 缺少代码驱动的特效

当前版本主要是：

```text
大图叠加 + 粉色线条 + 少量静态素材
```

这不等于正式特效。

《生命之母》需要的是代码和贴图结合：

```text
贴图负责复杂美术细节。
代码负责动态、生命感、连接感、节奏感、粒子流动。
```

### 必须补充的代码特效

```text
[必须实现]
- 飘带贝塞尔曲线摆动
- 飘带宽度变化
- 飘带内部流光
- 吸血光点沿飘带回流
- 花瓣粒子生成与消散
- 水晶碎光粒子
- 法阵旋转
- 光环呼吸缩放
- 机体核心亮度随血量变化
- 敌人被吸血时闪烁
- 大招展开时屏幕暗化/粉色滤镜
- 大招收束时粒子向中心吸入
- 最终爆发时白闪、冲击波、屏幕震动
```

## 1.3 飘带表现错误：不能做成粉色直线

当前版本最大问题之一是：

```text
生命飘带看起来像粉色激光线。
```

但设定要求是：

```text
生命飘带 = 半透明生命丝带 / 彩带 / 柔性光带
```

### 生命飘带应该具备的视觉特征

```text
- 有宽度，不是细线。
- 有柔软弧线，不是笔直激光。
- 有半透明丝绸质感。
- 有发光边缘。
- 有内部白粉色流光。
- 有轻微波动和摆动。
- 有端点装饰，例如水晶花、莲花、心形晶核。
- 有生命能量从敌人流回机体核心。
- 飘带周围有少量花瓣、碎晶、光粒。
```

## 1.4 大招连接过多导致卡顿和混乱

当前大招疑似同时连接全屏敌人，造成：

```text
- 连接线过多。
- 性能严重下降。
- 画面变成线团。
- 玩家看不懂技能目标。
- 主次不清晰。
```

### 正确连接逻辑

大招不需要视觉上连接所有敌人。  
应该采用“主连接目标 + 领域伤害”的混合逻辑。

```text
同一时刻最多显示 8 条主连接飘带。
主连接目标死亡后，再自动切换下一批。
未被主连接的敌人仍然受到领域持续伤害，但不显示完整飘带。
```

优先级：

```text
1. Boss
2. 精英怪
3. 高血量敌人
4. 离玩家较近的敌人
5. 普通小怪
```

## 3.1 绿幕素材预处理要求

所有纯绿色底图素材必须先处理为透明 PNG。

### Photoshop / Photopea 推荐流程

```text
1. 选择绿色背景。
2. 色彩范围选择 #00FF00 附近区域。
3. 选区扩展 1~2 px。
4. 羽化 0.3~0.8 px。
5. 删除背景。
6. 执行去色边 / Defringe 1 px。
7. 检查黑底、白底、游戏背景三种预览。
8. 导出 PNG-32。
```

### 程序预处理建议

如果程序侧自动抠绿：

```js
function chromaKeyGreen(imageData) {
  const data = imageData.data;

  for (let i = 0; i < data.length; i += 4) {
    const r = data[i];
    const g = data[i + 1];
    const b = data[i + 2];

    const isGreen =
      g > 160 &&
      g > r * 1.35 &&
      g > b * 1.35 &&
      r < 150 &&
      b < 150;

    if (isGreen) {
      data[i + 3] = 0;
      continue;
    }

    // 减少绿色溢色
    if (g > r && g > b) {
      data[i + 1] = Math.min(g, Math.max(r, b) + 20);
    }
  }

  return imageData;
}
```

# 4. 生命飘带系统重做方案

## 4.1 飘带不要用直线

错误写法：

```js
ctx.moveTo(player.x, player.y);
ctx.lineTo(enemy.x, enemy.y);
ctx.stroke();
```

这种只会得到“粉色激光线”。

---

## 4.2 正确飘带结构

每一条飘带建议由 5 层组成：

```text
Layer A：宽的半透明外发光
Layer B：丝带主体
Layer C：内部白色流光线
Layer D：沿路径移动的吸血光点
Layer E：端点花/晶体/命中粒子
```

---

## 4.3 飘带曲线绘制代码

```js
function drawLifeRibbon(ctx, from, to, time, options = {}) {
  const width = options.width ?? 10;
  const alpha = options.alpha ?? 0.85;
  const amp = options.amp ?? 50;
  const phase = options.phase ?? 0;
  const colorMain = options.colorMain ?? "rgba(255, 150, 220, 0.75)";
  const colorCore = options.colorCore ?? "rgba(255, 245, 255, 0.95)";

  const dx = to.x - from.x;
  const dy = to.y - from.y;
  const dist = Math.sqrt(dx * dx + dy * dy) || 1;

  const nx = -dy / dist;
  const ny = dx / dist;

  const midX = (from.x + to.x) / 2;
  const midY = (from.y + to.y) / 2;

  const wave = Math.sin(time * 3.2 + phase) * amp;

  const c1 = {
    x: midX + nx * wave,
    y: midY + ny * wave
  };

  ctx.save();
  ctx.globalCompositeOperation = "lighter";
  ctx.globalAlpha = alpha;
  ctx.lineCap = "round";
  ctx.lineJoin = "round";

  // 外层柔光
  ctx.shadowColor = "rgba(255, 120, 230, 1)";
  ctx.shadowBlur = 24;
  ctx.strokeStyle = "rgba(255, 120, 230, 0.22)";
  ctx.lineWidth = width * 3.2;
  ctx.beginPath();
  ctx.moveTo(from.x, from.y);
  ctx.quadraticCurveTo(c1.x, c1.y, to.x, to.y);
  ctx.stroke();

  // 主飘带层
  ctx.shadowBlur = 12;
  ctx.strokeStyle = colorMain;
  ctx.lineWidth = width;
  ctx.beginPath();
  ctx.moveTo(from.x, from.y);
  ctx.quadraticCurveTo(c1.x, c1.y, to.x, to.y);
  ctx.stroke();

  // 内部流光层
  ctx.shadowBlur = 6;
  ctx.strokeStyle = colorCore;
  ctx.lineWidth = Math.max(2, width * 0.25);
  ctx.beginPath();
  ctx.moveTo(from.x, from.y);
  ctx.quadraticCurveTo(c1.x, c1.y, to.x, to.y);
  ctx.stroke();

  ctx.restore();

  return c1;
}
```

## 4.4 吸血光点沿飘带回流

能量必须从敌人流向玩家，体现“吸血”。

```js
function bezierPoint(p0, p1, p2, t) {
  const a = 1 - t;
  return {
    x: a * a * p0.x + 2 * a * t * p1.x + t * t * p2.x,
    y: a * a * p0.y + 2 * a * t * p1.y + t * t * p2.y
  };
}

function drawDrainPulses(ctx, player, enemy, control, time, count = 3) {
  for (let i = 0; i < count; i++) {
    const t = (time * 0.7 + i / count) % 1;

    // 注意：从 enemy 到 player
    const p = bezierPoint(enemy, control, player, t);

    ctx.save();
    ctx.globalCompositeOperation = "lighter";
    ctx.globalAlpha = 0.9;
    ctx.shadowColor = "rgba(255, 230, 255, 1)";
    ctx.shadowBlur = 16;
    ctx.fillStyle = "rgba(255, 245, 255, 1)";
    ctx.beginPath();
    ctx.arc(p.x, p.y, 3.5 + Math.sin(time * 8 + i) * 1.2, 0, Math.PI * 2);
    ctx.fill();
    ctx.restore();
  }
}
```

# 5. 普通攻击重做要求

## 5.1 机制

```text
默认 4 条生命飘带。
随时间最多增加到 8 条。
单体时多条飘带可以叠加连接同一个目标。
自身吸血阶段持续扣血。
生命越低，吸血伤害越高。
10% 以下进入反转回血。
```

## 5.2 普通攻击视觉

```text
常态飘带：
- 宽度 6~10 px
- 半透明粉白
- 轻柔摆动
- 每条摆动相位不同
- 敌人端有小花印记
- 飘带内有 2~3 个回流光点
```

不要：

```text
- 所有飘带完全笔直
- 所有飘带同一条轨迹
- 线条过细
- 一次画太多杂乱线
```

# 6. 空格技能“生命共鸣”重做要求

## 6.1 机制

```text
持续 6 秒。
锁定 HP 最高敌人为主目标。
玩家到主目标显示一条粗主飘带。
```

```text
副飘带：
- 比主飘带细

- 看起来像生命网络

敌人标记：
- 被连接敌人头上出现小型生命花印
- 主目标标记更大、更亮
```

```js
function updateLifeResonance(player, enemies, dt) {
```

```js
function selectUltimateTargets(enemies) {
  return enemies
    .filter(e => e.alive)
    .sort((a, b) => getPriority(b) - getPriority(a))
    .slice(0, 8);
}
```

```js
function getPriority(enemy) {
  let score = 0;
  if (enemy.type === "boss") score += 10000;
  if (enemy.type === "elite") score += 5000;
  score += enemy.hp;
  score -= enemy.distanceToPlayer * 0.2;
  return score;
}
```

## 8.3 推荐代码

```js
function updateUltimateTargetPool(player, enemies) {
  const current = player.ultimate.targets.filter(e => e.alive);

  if (current.length >= 8) {
    player.ultimate.targets = current;
    return;
  }

  const candidates = enemies
    .filter(e => e.alive && !current.includes(e))
    .sort((a, b) => getPriority(b) - getPriority(a));

  while (current.length < 8 && candidates.length > 0) {
    current.push(candidates.shift());
  }

  player.ultimate.targets = current;
}
```

# 9. 大招领域伤害与连接伤害分离

为了减少画面混乱，伤害逻辑和视觉连接逻辑可以分开。

```text
视觉连接：最多 8 个主目标。
实际伤害：领域内所有敌人都受到 DOT。
```

```js
function updateUltimateDamage(player, enemies, dt) {
  const hpRate = player.hp / player.maxHp;

  let dps = player.attack * 2.2;
  if (hpRate < 0.3) dps *= 1.6;
  if (hpRate < 0.1) dps *= 2.2;

  let totalDamage = 0;

  for (const enemy of enemies) {
    if (!enemy.alive) continue;

    const damage = dps * dt;
    enemy.takeDamage(damage);
    totalDamage += damage;

    // 非主连接目标只显示弱命中特效
    if (!player.ultimate.targets.includes(enemy)) {
      enemy.showWeakDomainHit = true;
    }
  }

  healPlayer(player, totalDamage * 0.25);
}
```

这样既保留“全屏大招”的爽感，又不会强行画爆屏幕。

# 12. 粒子系统重做

## 12.1 粒子不是可选项

《生命之母》的高级感必须依靠粒子：

```text
花瓣
晶尘
光点
星屑
治疗光球
吸血回流点
```

## 12.2 通用粒子结构

```js
class Particle {
  constructor(config) {
    this.x = config.x;
    this.y = config.y;
    this.vx = config.vx ?? 0;
    this.vy = config.vy ?? 0;
    this.life = config.life ?? 1;
    this.maxLife = this.life;
    this.scale = config.scale ?? 1;
    this.rotation = config.rotation ?? 0;
    this.rotationSpeed = config.rotationSpeed ?? 0;
    this.sprite = config.sprite;
    this.alpha = config.alpha ?? 1;
    this.blend = config.blend ?? "lighter";
  }

  update(dt) {
    this.life -= dt;
    this.x += this.vx * dt;
    this.y += this.vy * dt;
    this.rotation += this.rotationSpeed * dt;
  }

  draw(ctx) {
    const t = Math.max(this.life / this.maxLife, 0);

    drawSprite(ctx, this.sprite, {
      x: this.x,
      y: this.y,
      scale: this.scale,
      rotation: this.rotation,
      alpha: this.alpha * t,
      blend: this.blend
    });
  }
}
```

## 12.3 粒子数量建议

```text
普通状态：20~40
小技能：50~80
暴走状态：80~120
大招持续：120~180
大招爆发瞬间：200 以内，爆发后快速减少
```

移动端低配模式：

```text

减少大贴图层数
```
