# 《生命之母》大招《众生胎宫》特效制作专用文档 V3

> 用途：直接提供给编程 AI / 程序员 / 特效实现人员  
> 目标：把《生命之母》的大招《众生胎宫》做成**正式可落地的游戏内特效**，而不是“贴图叠加预览版”  
> 适用项目：HTML / Canvas / WebGL / 2D STG 弹幕射击游戏  
> 核心要求：**分阶段演出 + 贴图与代码结合 + 飘带不是线 + 有回流感 + 有爆发感 + 性能可控**

---

# 0. 总目标

《众生胎宫》不是普通炸弹技能，也不是单纯的全屏伤害特效。  
它的核心视觉逻辑是：

```text
圣域展开
→ 生命飘带连接敌人
→ 吸收生命并回流到机体核心
→ 核心蓄能
→ 圣花爆发终结
→ 余辉消散
```

玩家在释放大招后，应该明显看到以下过程：

1. **战机开启生命圣域**
2. **最多 8 个主要敌人被生命飘带连接**
3. **生命能量沿飘带回流到战机**
4. **战机核心越来越亮**
5. **最终中央晶花 / 圣花完全绽放并爆发**
6. **冲击波、白闪、花瓣与晶体碎片扩散**
7. **领域淡出，不是突然消失**

---

# 1. 当前常见错误（必须避免）

## 1.1 不能把大图整张糊在屏幕上

错误表现：

- 整个屏幕被一张半透明花图压住
- 可以看到矩形边界
- 像“图层没抠干净”
- 看起来是素材预览，不像正式特效

正确要求：

- 花、法阵、光环、粒子都必须**分层**
- 只保留有效发光区域
- 按阶段分别出现
- 由代码控制缩放、旋转、透明度、粒子数量、闪烁、回流

---

## 1.2 飘带不能画成激光线 / 直线

错误表现：

- 几条细粉线连接敌人
- 像激光、射线、雷达线
- 没有丝带感、柔软感、生命感

正确要求：

- 飘带必须是**柔性丝带**
- 要有宽度、摆动、弧线、发光边缘、内部流光
- 要看起来像“生命彩带在抽取生命”，而不是扫射线

---

## 1.3 大招不能连接全屏所有敌人

错误表现：

- 每个敌人都被连线
- 线太多，画面乱
- 性能暴跌，明显卡顿

正确要求：

- **视觉上最多连接 8 个敌人**
- 当前 8 个死亡后再补下一批
- 其他敌人只吃领域 DOT，不强制显示完整连接

---

## 1.4 不能没有阶段感

错误表现：

- 大招从头到尾就是一堆贴图和线
- 看不出启动、持续、收束、爆发的区别

正确要求：

- 必须拆成多个阶段
- 每个阶段的贴图、光效、粒子和运动逻辑不同

---

## 3.4 动画逻辑

### 启动法阵
- scale: `0.3 → 1.4`
- alpha: `0 → 0.85`
- rotation: 缓慢逆时针
- 时长：0.5 秒

### 花苞开花
- 方案 A：多张图切换（花苞 → 半开 → 盛开）
- 方案 B：单图缩放 + alpha 变化（不推荐，层次差）

### 光环
- 从透明到明亮
- 同时轻微扩大
- 有轻微呼吸感

### 启动冲击环
- 从战机中心快速扩散 1 次
- alpha 迅速减弱
- 作为“启动反馈”

## 3.5 代码伪代码

```js
function renderUltimateStart(ctx, player, t) {
  // t 范围 0~1
  const ease = easeOutCubic(t);

  drawScreenTint(ctx, `rgba(255,220,245,${0.12 * ease})`);

  drawSprite(ctx, assets.ultimate_start_circle_01, {
    x: player.x,
    y: player.y,
    scale: 0.3 + ease * 1.1,
    rotation: -t * 0.8,
    alpha: 0.85 * ease,
    blend: "lighter"
  });

  drawSprite(ctx, assets.ultimate_start_halo_01, {
    x: player.x,
    y: player.y - 45,
    scale: 0.5 + ease * 0.5,
    rotation: t * 0.5,
    alpha: 0.75 * ease,
    blend: "lighter"
  });

  // 花苞切帧
  const frame = t < 0.33 ? assets.ultimate_start_bud_01
              : t < 0.66 ? assets.ultimate_start_bud_02
              : assets.ultimate_start_bloom_01;

  drawSprite(ctx, frame, {
    x: player.x,
    y: player.y,
    scale: 0.5 + ease * 0.5,
    alpha: 1,
    blend: "lighter"
  });
}
```

## 4.4 飘带的正确做法

### 不能做法
- `lineTo` 直接画线
- 只用一根粉色笔划
- 没有宽度变化，没有内流光

### 正确做法
每条飘带由 5 层构成：

```text
1. 外发光层（宽、淡）
2. 主丝带层（有宽度）
3. 内流光层（较细白线）
4. 光点回流层（小球/粒子移动）
5. 端点装饰层（玩家端 / 敌人端）
```

---

## 4.5 飘带绘制建议

### 贝塞尔曲线绘制

```js
function drawLifeRibbon(ctx, from, to, time, options = {}) {
  const width = options.width ?? 12;
  const glow = options.glow ?? 24;
  const alpha = options.alpha ?? 0.85;
  const amp = options.amp ?? 42;
  const phase = options.phase ?? 0;

  const dx = to.x - from.x;
  const dy = to.y - from.y;
  const dist = Math.sqrt(dx * dx + dy * dy) || 1;

  const nx = -dy / dist;
  const ny = dx / dist;

  const mx = (from.x + to.x) / 2;
  const my = (from.y + to.y) / 2;

  const wave = Math.sin(time * 2.8 + phase) * amp;

  const ctrl = {
    x: mx + nx * wave,
    y: my + ny * wave
  };

  ctx.save();
  ctx.globalCompositeOperation = "lighter";
  ctx.globalAlpha = alpha;
  ctx.lineCap = "round";
  ctx.lineJoin = "round";

  // 外发光
  ctx.shadowColor = "rgba(255,150,230,1)";
  ctx.shadowBlur = glow;
  ctx.strokeStyle = "rgba(255,160,230,0.25)";
  ctx.lineWidth = width * 2.8;
  ctx.beginPath();
  ctx.moveTo(from.x, from.y);
  ctx.quadraticCurveTo(ctrl.x, ctrl.y, to.x, to.y);
  ctx.stroke();

  // 主丝带
  ctx.shadowBlur = 12;
  ctx.strokeStyle = "rgba(255,185,240,0.72)";
  ctx.lineWidth = width;
  ctx.beginPath();
  ctx.moveTo(from.x, from.y);
  ctx.quadraticCurveTo(ctrl.x, ctrl.y, to.x, to.y);
  ctx.stroke();

  // 内层白光
  ctx.shadowBlur = 8;
  ctx.strokeStyle = "rgba(255,255,255,0.88)";
  ctx.lineWidth = Math.max(2, width * 0.25);
  ctx.beginPath();
  ctx.moveTo(from.x, from.y);
  ctx.quadraticCurveTo(ctrl.x, ctrl.y, to.x, to.y);
  ctx.stroke();

  ctx.restore();

  return ctrl;
}
```

## 4.6 吸血回流光点

### 目标
明确表达“生命从敌人流向战机”。

### 推荐做法
- 每条飘带上挂 2~4 个光点
- 光点从敌人端向玩家端移动
- 光点颜色偏粉白 / 乳白 / 淡金
- 可以同时生成少量小尾迹粒子

```js
function bezierPoint(p0, p1, p2, t) {
  const u = 1 - t;
  return {
    x: u * u * p0.x + 2 * u * t * p1.x + t * t * p2.x,
    y: u * u * p0.y + 2 * u * t * p1.y + t * t * p2.y
  };
}

function drawRibbonFlowDots(ctx, enemy, ctrl, player, time, count = 3) {
  for (let i = 0; i < count; i++) {
    const t = (time * 0.75 + i / count) % 1; // 从 enemy → player
    const p = bezierPoint(enemy, ctrl, player, t);

    ctx.save();
    ctx.globalCompositeOperation = "lighter";
    ctx.shadowColor = "rgba(255,230,255,1)";
    ctx.shadowBlur = 16;
    ctx.fillStyle = "rgba(255,245,255,0.95)";
    ctx.globalAlpha = 0.85;
    ctx.beginPath();
    ctx.arc(p.x, p.y, 3.5 + Math.sin(time * 7 + i) * 0.8, 0, Math.PI * 2);
    ctx.fill();
    ctx.restore();
  }
}
```

## 4.7 法阵和光环

### 地面大法阵
- 低透明度
- 持续缓慢旋转
- 不能太亮，否则压过战机和飘带
- 建议 2 层：底层法阵 + 上层薄光环

### 上层光环
- 浮在战机上方或战机周围
- 比地面法阵更亮
- 可轻微呼吸

## 5.5 向心粒子逻辑

```js
function updateInwardParticles(particles, center, dt) {
  for (const p of particles) {
    const dx = center.x - p.x;
    const dy = center.y - p.y;
    const len = Math.sqrt(dx * dx + dy * dy) || 1;

    p.vx += (dx / len) * 160 * dt;
    p.vy += (dy / len) * 160 * dt;

    p.x += p.vx * dt;
    p.y += p.vy * dt;

    p.alpha *= 0.992;
  }
}
```

## 6.4 动画逻辑

### 白闪
- 短促
- 强
- 不宜太长
- 建议 0.08s ~ 0.18s

### 冲击波
- scale: `0.4 → 4.5`
- alpha: `0.9 → 0`
- 可做 1~2 层，内环和外环速度不同

### 最大圣花
- 中心出现
- 快速放大
- 轻微旋转
- 随后开始淡出

### 爆散粒子
- 花瓣向外飞
- 碎晶向外飞
- 小光点四散
- 随时间渐隐

function renderUltimateBurst(ctx, player, t) {
  const e = easeOutCubic(t);

  // 屏幕白闪

    blend: "lighter"

    alpha: 0.9 * (1 - t),
    blend: "lighter"
  });

  screenShake(0.25, 8);
}

# 9. 贴图与代码职责分工

这是最关键的落地原则。

## 9.1 贴图负责什么

```text
复杂形状和细节：
- 圣花
- 法阵纹样
- 光环外形
- 心核
- 飘带端点
- 敌人生命印记
- 花瓣、碎晶、小装饰
- 冲击波底图
```

## 9.2 代码负责什么

```text
动态变化：
- 飘带曲线
- 飘带摆动
- 飘带宽度变化
- 飘带流光
- 回流光点移动
- 粒子生成 / 飞散 / 吸入
- 法阵旋转
- 光环呼吸
- 屏幕暗化 / 白闪
- 冲击波扩张
- 屏幕震动
- 阶段切换
```

## 9.3 总结原则

```text
不要试图用贴图替代动态。
不要试图用一张大图完成整个大招。
必须：贴图做美术，代码做生命。
```

# 10. 性能优化指导

## 10.1 连接目标上限

```text
视觉主连接目标最多 8 个
```

大招不要连全屏所有敌人。

---

## 10.2 粒子数量建议

```text
启动阶段：30~50
持续阶段：80~140
收束阶段：100~160
爆发瞬间：160~220
余辉阶段：20~40
```

移动端低配模式可减半。

---

## 10.3 Canvas / WebGL 注意点

### Canvas
- `shadowBlur` 不要无限开大
- 法阵可预渲染
- 粒子使用对象池

### WebGL
- 粒子推荐批量渲染
- 法阵和光环可做 instancing 或统一 shader
- 大量半透明叠加要控制 overdraw

## 11.2 状态切换

```js
function updateUltimate(player, dt) {
  if (!player.ultimate.active) return;

  player.ultimate.timer += dt;

  if (player.ultimate.timer < 0.6) {
    player.ultimate.state = UltimateState.START;
  } else if (player.ultimate.timer < 4.5) {
    player.ultimate.state = UltimateState.DRAIN;
  } else if (player.ultimate.timer < 5.3) {
    player.ultimate.state = UltimateState.GATHER;
  } else if (player.ultimate.timer < 6.0) {
    player.ultimate.state = UltimateState.BURST;
  } else if (player.ultimate.timer < 6.5) {
    player.ultimate.state = UltimateState.FADEOUT;
  } else {
    player.ultimate.active = false;
    player.ultimate.state = UltimateState.IDLE;
    player.ultimate.timer = 0;
  }
}
```
