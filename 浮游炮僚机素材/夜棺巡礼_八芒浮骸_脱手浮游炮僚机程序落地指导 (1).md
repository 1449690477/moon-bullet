# 夜棺巡礼·八芒浮骸 —— 脱手浮游炮僚机落地制作指导 MD

> 项目类型：HTML5 / Canvas / PixiJS / Phaser / WebGL 2D STG 弹幕射击游戏  
> 适用风格：哥特萝莉、神话暗黑、圣窗玻璃、黑银荆棘、紫蓝幽能、暗夜教堂  
> 僚机定位：**不跟随主角的脱手自动攻击型浮游炮阵列**  
> 核心单位：8 枚同外观浮游炮  
> 重要原则：**浮游炮外观只有一种，不允许因为模式切换随便换外形。所有模式变化只通过姿态、展开部件、核心亮度、拖尾、激光、粒子和音效表现。**

---

# 0. 制作目标总览

本僚机不是普通跟随型僚机，而是一个释放后独立作战的浮游炮阵列。

它的战斗逻辑分为两种形态：

```text
默认形态：群体冲撞模式
Q 切换：浮游炮激光模式
```

暴走状态下，两种形态分别强化：

```text
默认暴走：超限冲撞模式
Q 暴走：超限激光绞杀模式
```

最终局内表现应当是：

- 8 枚哥特圣窗风格浮游炮脱手离场。
- 默认形态下，8 枚浮游炮组成机群，在敌群之间来回穿梭冲撞。
- 每次冲撞留下暗蓝紫拖尾，命中敌人产生 AOE 爆裂和减速领域。
- 按 Q 后，8 枚浮游炮重新列阵，锁定血量最高或威胁最高的敌人，从多个角度发射激光束。
- 暴走后，推进器、核心和边缘暗紫幽能超载，冲撞拖尾变粗、变深、变长；激光模式下光束变粗、发射频率变高、命中特效更强。

---

## 1.3 外观统一规则

这是最重要的制作要求：

> **所有模式下浮游炮本体必须使用同一套外观。**

不能出现：

- 冲撞模式一个外观
- 激光模式换成另一个外观
- 暴走模式变成完全不同的机体
- 子弹贴图里的浮游炮和本体贴图长得不一样

正确做法是：

```text
同一个浮游炮本体贴图
+ 不同朝向
+ 不同发光层
+ 不同拖尾
+ 不同激光
+ 不同核心亮度
+ 不同外缘暗焰
```

也就是说，形态变化是**特效变化**，不是换模型。

# 4. 默认形态：群体冲撞模式

## 4.1 玩法描述

8 枚浮游炮组成独立机群，不跟随主角，而是在战场中自动寻找敌人。

行为逻辑：

```text
寻找敌群
→ 选择穿梭路径
→ 高速冲刺
→ 经过敌人时造成 AOE 伤害
→ 命中区域产生减速效果
→ 机群转向
→ 再次冲撞
```

---

## 4.2 视觉表现

普通冲撞状态下：

- 浮游炮前端朝向运动方向。
- 后方拖出蓝紫幽能尾流。
- 尾流不是白色棒子，而是暗紫能量流。
- 每次经过敌群时产生爆点。
- 命中区域留下短时间减速残留圈。

正确视觉层次：

```text
后方残影
+ 暗紫外焰拖尾
+ 蓝紫主尾流
+ 白蓝核心流线
+ 浮游炮本体
+ 前端破空亮点
+ 命中爆闪
+ 地面/空中减速残留
```

---

## 4.3 错误表现禁止

不要做成：

- 8 个浮游炮排队平移
- 白色棒状拖尾
- 半透明三角形罩子
- 只有贴图没有残影
- 命中后直接消失
- 冲撞轨迹没有弯曲

---

## 4.4 冲撞路径设计

建议使用“目标点 → 冲刺 → 折返”的行为。

### 路径类型

| 类型 | 用途 |
|---|---|
| 直线穿刺 | 高速穿过一排敌人 |
| 弧线切入 | 从侧面扫过敌群 |
| S 型穿梭 | 大范围清小怪 |
| 交叉冲撞 | 多枚浮游炮交错穿过敌群 |
| 八芒阵冲撞 | 8 枚从不同方向向中心冲撞 |

---

## 4.5 目标选择

```js
function selectDashTarget(enemies) {
  // 优先选择敌人密集区域，而不是单个敌人
}
```

推荐评分：

```text
score =
敌人数量 * 3
+ 精英怪权重 * 5
+ Boss 权重 * 3
+ 距离玩家适中加成
- 已被刚冲撞过的区域惩罚
```

---

## 4.6 AOE 与减速

每次冲撞经过敌人或目标点时：

```text
造成 AOE 伤害
+ 施加减速
+ 生成冲撞爆点
+ 生成暗紫减速残留纹理
```

推荐数值：

| 项目 | 普通冲撞 |
|---|---|
| AOE 半径 | 45~80 px |
| 减速 | 25%~40% |
| 减速持续 | 1.2~2.0 秒 |
| 单次冲撞伤害 | 中等 |
| 冲撞频率 | 每组 0.6~1.2 秒 |

# 6. Q 切换形态：浮游炮激光模式

## 6.1 玩法描述

按 Q 后，8 枚浮游炮从群体冲撞模式切换为固定/半固定激光阵列模式。

核心逻辑：

```text
停止自由穿梭
→ 8 枚浮游炮重组成环形或半环阵列
→ 自动锁定血量最高 / 威胁最高的敌人
→ 每枚浮游炮从不同角度瞄准目标
→ 发射持续激光束
→ 对单体造成高频伤害
```

---

## 6.2 外观规则

Q 模式下**浮游炮本体不能换外形**。

正确变化是：

- 机体旋转朝向目标。
- 核心亮度上升。
- 前端炮口出现发射闪光。
- 圣窗内部出现紫蓝能量流动。
- 机体边缘可以出现很薄的黑紫光晕。
- 可以叠加“核心强化 overlay”，但不能改本体轮廓。

---

## 6.3 阵列方式

推荐三种阵列：

### A. 环形锁定阵

8 枚浮游炮围绕目标形成圆环。

```text
       o
   o       o
 o    Boss   o
   o       o
       o
```

适合 Boss 对单。

---

### B. 半圆压制阵

8 枚浮游炮分布在玩家前方或敌人周围半圆。

```text
o o o o Boss o o o o
```

适合横向压制。

---

### C. 多角度绞杀阵

8 枚从不同角度射向同一个敌人。

```text
\  |  /
 - Boss -
/  |  \
```

最符合“无死角打击”。

---

## 6.4 目标选择

激光模式目标必须是单体高威胁目标。

优先级：

1. Boss
2. 精英怪
3. 当前血量最高敌人
4. 攻击力最高敌人
5. 距离玩家较近且威胁高的敌人

示例：

```js
function selectLaserTarget(enemies) {
  return enemies
    .filter(e => e.alive)
    .sort((a, b) => getThreatScore(b) - getThreatScore(a))[0];
}

function getThreatScore(e) {
  return e.hp * 0.4
       + e.maxHp * 0.3
       + (e.isBoss ? 10000 : 0)
       + (e.isElite ? 3000 : 0)
       + e.damage * 10;
}
```

---

## 6.5 激光伤害

激光不是单次伤害，而是持续 tick。

| 项目 | 普通激光 |
|---|---|
| Tick 间隔 | 0.05~0.1 秒 |
| 每 Tick 伤害 | 低~中 |
| 总输出 | 对单高 |
| 命中特效 | 高频小爆点 |
| 是否穿透 | 可选，主要锁单体 |
| 光束宽度 | 中等偏细 |

# 8. 贴图如何作为辅助素材使用

---

## 8.1 贴图不是最终特效

贴图负责：

- 外观轮廓
- 基础纹理
- 哥特圣窗细节
- 基础拖尾纹理
- 基础爆炸形状
- 基础光束形状

代码负责：

- 动态发光
- 粒子
- 残影
- 曲线拖尾
- 光束脉冲
- 命中震动
- 屏幕反馈
- 模式转换动画

不要只做：

```js
drawImage(texture, x, y);
```

要做：

```text
贴图本体
+ 程序光效
+ 程序粒子
+ 程序残影
+ 程序冲击波
+ 程序屏幕反馈
```

---

## 8.2 浮游炮本体分层

推荐绘制层：

```text
Layer 0：阴影 / 暗雾背光
Layer 1：后方暗紫外焰
Layer 2：浮游炮本体
Layer 3：圣窗核心发光
Layer 4：边缘金属高光
Layer 5：暴走 overlay
Layer 6：粒子 / 电弧
```

绘制顺序示例：

```js
drawDarkBackGlow(drone);
drawSprite(droneBody, drone.x, drone.y, { rotation: drone.angle });
drawCoreGlow(drone);
drawEdgeHighlight(drone);
if (overdrive) drawOverdriveOverlay(drone);
drawDroneParticles(drone);
```

---

## 8.3 拖尾贴图正确使用

拖尾贴图不要像棒子一样固定贴在后面。

正确做法：

- 用贴图做基础纹理。
- 用 history 轨迹决定拖尾曲线。
- 用代码绘制核心线。
- 用贴图叠在局部区域增加纹理。
- 转向时，拖尾根据历史路径弯曲。

错误：

```text
每把飞剑后面贴一根固定白条
```

正确：

```text
每把浮游炮根据自身速度历史生成暗紫能量尾流
```

# 9. 冲撞拖尾代码实现

---

## 9.1 每枚浮游炮保存历史轨迹

```js
class DroneUnit {
  constructor(index, x, y) {
    this.index = index;
    this.x = x;
    this.y = y;
    this.vx = 0;
    this.vy = 0;
    this.angle = 0;
    this.history = [];
    this.maxHistory = 12;
    this.state = 'dash';
    this.dashCooldown = 0;
  }

  update(dt) {
    this.history.unshift({ x: this.x, y: this.y });

    if (this.history.length > this.maxHistory) {
      this.history.pop();
    }

    this.x += this.vx * dt;
    this.y += this.vy * dt;

    if (Math.abs(this.vx) + Math.abs(this.vy) > 0.01) {
      this.angle = Math.atan2(this.vy, this.vx);
    }
  }
}
```

---

## 9.2 多层拖尾绘制

```js
function drawDroneDashTrail(ctx, drone, isOverdrive) {
  if (drone.history.length < 2) return;

  ctx.save();
  ctx.globalCompositeOperation = 'lighter';
  ctx.lineCap = 'round';
  ctx.lineJoin = 'round';

  const mul = isOverdrive ? 1.6 : 1.0;

  drawTrailLayer(ctx, drone.history, 46 * mul, 'rgba(20, 0, 40, 0.22)');
  drawTrailLayer(ctx, drone.history, 30 * mul, 'rgba(90, 30, 180, 0.35)');
  drawTrailLayer(ctx, drone.history, 16 * mul, 'rgba(60, 180, 255, 0.45)');
  drawTrailLayer(ctx, drone.history, 5 * mul, 'rgba(240, 255, 255, 0.95)');

  ctx.restore();
}

function drawTrailLayer(ctx, points, baseWidth, color) {
  ctx.strokeStyle = color;

  for (let i = 0; i < points.length - 1; i++) {
    const p1 = points[i];
    const p2 = points[i + 1];

    const t = 1 - i / points.length;
    ctx.globalAlpha = t * t;
    ctx.lineWidth = baseWidth * t;

    ctx.beginPath();
    ctx.moveTo(p1.x, p1.y);

    const midX = (p1.x + p2.x) * 0.5;
    const midY = (p1.y + p2.y) * 0.5;

    ctx.quadraticCurveTo(p1.x, p1.y, midX, midY);
    ctx.stroke();
  }

  ctx.globalAlpha = 1;
}
```

## 11.2 激光绘制结构

每条激光分为 5 层：

```text

Layer 2：紫色主光束
```

## 11.3 光束必须从炮口发射

这是重点：

> 激光起点必须是浮游炮前端炮口，而不是机体中心，也不是随机位置。

计算炮口：

```js
function getDroneMuzzlePosition(drone) {
  const dx = Math.cos(drone.angle);
  const dy = Math.sin(drone.angle);

  return {
    x: drone.x + dx * drone.muzzleOffset,
    y: drone.y + dy * drone.muzzleOffset
  };
}
```

`muzzleOffset` 根据贴图大小调整，例如 42~70 px。

---

## 11.4 激光对准目标

```js
function aimDroneAtTarget(drone, target) {
  const dx = target.x - drone.x;
  const dy = target.y - drone.y;
  drone.angle = Math.atan2(dy, dx);
}
```

## 11.5 绘制激光

```js
function drawDroneLaser(ctx, drone, target, isOverdrive, time) {
  const muzzle = getDroneMuzzlePosition(drone);

  const x1 = muzzle.x;
  const y1 = muzzle.y;
  const x2 = target.x;
  const y2 = target.y;

  const pulse = 1 + Math.sin(time * 24 + drone.index) * 0.06;
  const widthMul = isOverdrive ? 1.65 : 1.0;

  ctx.save();
  ctx.globalCompositeOperation = 'lighter';
  ctx.lineCap = 'round';

  drawLaserLayer(ctx, x1, y1, x2, y2, 34 * widthMul * pulse, 'rgba(20, 0, 45, 0.22)');
  drawLaserLayer(ctx, x1, y1, x2, y2, 22 * widthMul * pulse, 'rgba(120, 40, 255, 0.35)');
  drawLaserLayer(ctx, x1, y1, x2, y2, 10 * widthMul * pulse, 'rgba(80, 200, 255, 0.55)');
  drawLaserLayer(ctx, x1, y1, x2, y2, 3.5 * widthMul * pulse, 'rgba(255, 255, 255, 0.95)');

  ctx.restore();

  drawLaserMuzzleFlash(ctx, x1, y1, isOverdrive, time);
  drawLaserHitPoint(ctx, x2, y2, isOverdrive, time);
}

function drawLaserLayer(ctx, x1, y1, x2, y2, width, color) {

}
```

```js
function drawLaserTextureOverlay(ctx, texture, x1, y1, x2, y2, time, alpha) {
  const angle = Math.atan2(y2 - y1, x2 - x1);
  const length = Math.hypot(x2 - x1, y2 - y1);
```

## 15.2 粒子对象

```js
class Particle {
  constructor(x, y, vx, vy, life, size, color) {
    this.x = x;
    this.y = y;
    this.vx = vx;
    this.vy = vy;
    this.life = life;
    this.maxLife = life;
    this.size = size;
    this.color = color;
  }

  update(dt) {
    this.x += this.vx * dt;
    this.y += this.vy * dt;
    this.life -= dt;
  }

  draw(ctx) {
    const t = Math.max(this.life / this.maxLife, 0);
    ctx.save();
    ctx.globalCompositeOperation = 'lighter';
    ctx.globalAlpha = t * t;
    ctx.fillStyle = this.color;
    ctx.beginPath();
    ctx.arc(this.x, this.y, this.size * t, 0, Math.PI * 2);
    ctx.fill();
    ctx.restore();
  }
}
```

```js
const particlePool = [];
const particles = [];

function spawnParticle(config) {
  let p = particlePool.pop();

  if (!p) {
    p = new Particle(0, 0, 0, 0, 1, 1, '#fff');
  }

  Object.assign(p, config);
  particles.push(p);
}

function recycleDeadParticles() {
  for (let i = particles.length - 1; i >= 0; i--) {
    if (particles[i].life <= 0) {
      particlePool.push(particles[i]);
      particles.splice(i, 1);
    }
  }
}
```
